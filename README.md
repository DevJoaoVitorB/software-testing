## 🧪 Laboratório de Testes de Software: Algoritmos & Serviços
Este repositório é um ambiente de treinamento dedicado ao domínio de testes automatizados e garantia de qualidade. O objetivo principal é explorar como transformar requisitos técnicos em suítes de testes robustas, utilizando a validação de algoritmos complexos e lógica de serviços como base de estudo.

### 🎯 Objetivos do Repositório
O projeto foi estruturado para o aprendizado prático de como entender os diferentes tipos de testes e como automatizá-los com eficiência:

- Validação de Algoritmos e Serviços: Aplicação de testes unitários para garantir que a lógica de processamento e as regras de serviço funcionem exatamente como o esperado.
- Data-Driven Testing (DDT): Uso de matrizes de dados para automatizar a validação de múltiplos cenários (sucesso, erro e casos de borda), reduzindo a repetição de código e aumentando a confiabilidade.
- Fundamentos de Automação: Entender o fluxo de configuração de ferramentas de teste (Jest) para obter feedback imediato sobre a saúde e integridade do software.
- Análise de Cobertura (Coverage): Prática de interpretar relatórios para garantir que todas as ramificações de decisões e caminhos críticos dos serviços foram exercitados.

### 🛠️ Tecnologias Utilizadas

1. TypeScript: Garantia de segurança e clareza através da tipagem estática nos contratos de entrada e saída.
2. Jest: Framework principal para execução, organização e asserção da suíte de testes.
3. ts-jest: Integração técnica para execução direta de testes em arquivos TypeScript.
